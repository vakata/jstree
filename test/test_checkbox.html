Getting Started

The required HTML

jstree handles all additional elements dynamically. However, you will need to start the tree structure with the following HTML.

  • Root node 1
    • Child node 1
    • Child node 2
  • Root node 2 
<div id="demo1" class="demo" style="height:100px;">
  <ul>
    <li id="phtml_1">
      <a href="#">Root node 1</a>
      <ul>
        <li id="phtml_2">
          <a href="#">Child node 1</a>
        </li>
        <li id="phtml_3">
          <a href="#">Child node 2</a>
        </li>
      </ul>
    </li>
    <li id="phtml_4">
      <a href="#">Root node 2</a>
    </li>
  </ul>
</div>

<script src="//ajax.googleapis.com/ajax/libs/jquery/1.10.1/jquery.min.js"></script>
<script src="../src/jstree.js"></script>
<script src="../src/jstree.checkbox.js"></script>
<script>
jQuery("#demo1").jstree({
    checkbox : {
      real_checkboxes : true,
      real_checkboxes_names : function (n)
      {
        return [("checkbox_" + (n[0].id)), n[0].id];
      }
    }
    ,plugins : ["checkbox"]
  });
</script>

Options

Options can be passed directly to jstree via JavaScript.

Name type default description
real_checkboxes boolean false If set to true real hidden checkboxes will be created for each element, so if the tree is part of a form, checked nodes will be submitted automatically. By default the name of the checkbox is "check_" + the ID of the LI element and the value is 1, this can be changed using the real_checkboxes_names config option.
real_checkboxes_names function function(n){return [("c"+(n[0].id)),n[0].id]} If real checkboxes are used this function is invoked in the current tree's scope for each new checkbox that is created. It receives a single argument - the node that will contain the checkbox. The function must return an array consisting of two values - the name for the checkbox and the value for the checkbox.
